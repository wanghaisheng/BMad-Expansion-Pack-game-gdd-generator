# Analyze Game Design Structure

## Purpose
Perform comprehensive structural analysis of a game design to identify strengths, weaknesses, and improvement opportunities.

## Process

### 1. Identify Design Structure Type
- Linear progression
- Hub-based structure
- Open world design
- Metroidvania progression
- Branching narrative paths
- Procedural generation
- Level-based progression
- Sandbox structure

### 2. Map Key Design Points
- **Opening**: Tutorial, hook, core mechanic introduction
- **Inciting Challenge**: What drives the player forward?
- **First Major Milestone**: What locks in the core gameplay loop?
- **Midpoint Revelation**: What new mechanics/story elements emerge?
- **Climactic Challenge**: What raises difficulty to maximum?
- **Final Boss/Challenge**: How does central conflict resolve?
- **Resolution**: What post-game content or closure exists?

### 3. Analyze Progression Pacing
- Level difficulty distribution
- Skill learning curve
- Power progression balance
- Challenge escalation timing
- Rest/exploration moment placement

### 4. Evaluate Mechanic Integration
- Track all introduced mechanics
- Verify each has meaningful development
- Identify underutilized mechanics
- Find overpowered combinations
- Check tutorial-to-mastery progression

### 5. Assess System Interconnection
- List all game systems
- Track system interactions
- Evaluate balance between systems
- Check thematic reinforcement
- Identify system conflicts

### 6. Generate Report
Create design analysis report including:
- Progression flow diagram
- Difficulty curve chart
- Problem areas identification
- Suggested improvements
- Alternative design approaches

## Output
Comprehensive game design analysis with actionable recommendations
