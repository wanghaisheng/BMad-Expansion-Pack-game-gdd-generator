# Game Design Patterns Reference

## Core Gameplay Patterns

### Progression Patterns
- **Linear Progression**: Sequential unlocking of content and abilities
- **Branching Progression**: Multiple paths with player choice
- **Skill Trees**: Hierarchical ability unlocking systems
- **Prestige Systems**: Reset progression for meta-progression rewards
- **Seasonal Progression**: Time-limited progression tracks

### Engagement Patterns
- **Feedback Loops**: Systems that reinforce player behavior
- **Flow State Design**: Balancing challenge and skill progression
- **Variable Ratio Rewards**: Unpredictable reward timing for engagement
- **Social Proof**: Showing other players' achievements and progress
- **Loss Aversion**: Making players reluctant to lose progress

### Retention & Behavioral Patterns
- **3B Framework**: Behavior, Barriers, Benefits for retention goals
- **7-Day Behavior Plan**: Early-session objectives and pacing checkpoints
- **Habit Formation Loops**: Trigger → Action → Reward → Investment

### Emotional Pacing Patterns
- **Core Thrill Loop vs Meta Loop**: Micro feedback vs macro progression
- **Dopamine Pacing**: Fast peaks, clear payoff timing, short feedback cycles
- **Core Experience Purification**: Reduce friction to amplify core flow

### Monetization Patterns
- **Freemium**: Free base game with premium upgrades
- **Battle Pass**: Seasonal progression with premium tiers
- **Gacha/Loot Boxes**: Random reward systems
- **Cosmetic Monetization**: Visual customization purchases
- **Convenience Monetization**: Time-saving purchases

## Platform-Specific Patterns

### Mobile Game Patterns
- **Session-Based Design**: Short, repeatable play sessions
- **Energy Systems**: Time-gated progression mechanics
- **Daily Rewards**: Login incentive systems
- **Push Notifications**: Re-engagement through notifications
- **One-Handed Play**: Interface design for mobile ergonomics

### PC/Console Patterns
- **Achievement Systems**: Long-term goal tracking
- **Modding Support**: User-generated content systems
- **Difficulty Options**: Scalable challenge systems
- **Save Anywhere**: Flexible save/load systems
- **Controller Optimization**: Input method optimization

### VR/AR Patterns
- **Comfort-First Design**: Motion sickness prevention
- **Spatial Interaction**: 3D space utilization
- **Presence Reinforcement**: Immersion-enhancing mechanics
- **Physical Safety**: Real-world obstacle awareness
- **Gesture Recognition**: Natural movement interpretation

## Anti-Patterns to Avoid

### Player Experience Anti-Patterns
- **Pay-to-Win**: Monetary advantage in competitive contexts
- **Dark Patterns**: Manipulative design practices
- **Grinding Walls**: Excessive repetition requirements
- **Complexity Creep**: Overwhelming feature accumulation
- **Tutorial Hell**: Overly long or restrictive tutorials

### Technical Anti-Patterns
- **Performance Neglect**: Ignoring optimization requirements
- **Platform Inconsistency**: Different experiences across platforms
- **Accessibility Afterthought**: Late-stage accessibility consideration
- **Data Privacy Violations**: Inappropriate data collection
- **Security Vulnerabilities**: Inadequate security measures

## Pattern Selection Guidelines

### Choosing Appropriate Patterns
1. **Audience Analysis**: Match patterns to target player preferences
2. **Platform Constraints**: Consider technical and cultural limitations
3. **Business Model Alignment**: Ensure patterns support monetization goals
4. **Competitive Differentiation**: Use patterns to create unique value
5. **Accessibility Requirements**: Include inclusive design patterns

### Pattern Combination Strategies
1. **Synergistic Combinations**: Patterns that reinforce each other
2. **Balanced Implementation**: Avoiding pattern conflicts
3. **Progressive Complexity**: Introducing patterns gradually
4. **Cultural Sensitivity**: Adapting patterns for different markets
5. **Performance Considerations**: Ensuring patterns don't impact performance

This reference guide helps game designers select and implement proven design patterns while avoiding common pitfalls and ensuring inclusive, engaging player experiences.
